NDK介绍

什么是NDK开发

  1. Android NDK：Android NDK 是一套允许您使用 C 和 C++ 等语言，以原生代码实现部分应用的工具集
  2. 用来做Native开发或者叫so库开发
  3. .so文件
  4. 官方网站：https://developer.android.com/ndk

安装NDK

  • NDK是和SDK对应的，是对SDK的一个扩展，比如编译so文件，编译arm文件，需要用到这个NDK

方式1：通过SDK manager.exe安装(这种方法应该需要翻墙)

方式2：已安装Android Studio的情况下，可以通过Studio来进行安装

  • 此方法安装好之后，ndk-build在sdk目录/ndk-bundle\build目录下
  • 检测是否配置好环境： 终端输入：ndk-build

使用NDK

  1. 在ndk根路径下\toolchains\arm-linux-androideabi-4.9\prebuilt\windows-x86_64\bin这个路径里面，有操作elf文件的各种命令， 将此路径放入path环境变量，即可用
  2. cd进入c文件和mk文件的文件夹的父级目录，比如下面例子，进入jni目录下
  3. 输入ndk-build， 即可编译成功 ，得到ELF可执行程序

NDK开发

1. 创建android项目

2. java代码中引入native相关代码

3. 生成 JNI 样式的标头文件 /.h文件

4. android根目录下创建JNI目录，并将第三步生成的文件移入此文件夹

5. 创建和第三步生成的.h文件名字相同的.c文件

  • 注意
    • c文件的头文件选择第三步创建的.h文件的名字
    • 定义参数的名字
    • 编写逻辑时注意，要使用变量或者方法，需要使用jni方法，比如使用字符串类型，需要这么用，且不能使用中文

6. 按照编译可执行程序的方法将c编译成so文件[参考下面]

7. 在java代码中引用调用so文件

  1. 在Java中申明一个Native方法， 如securityCheck方法
  2. 用System.loadLibrary()加载so库
  • System.loadLibrary()加载so文件流程
    • 先读取so文件的.init_array段，.init_array段里面做的是变量的初始化操作
    • 再执行JNI_OnLoad函数，JNI_ONLoad是.so文件的初始函数
    • 然后调用具体的native方法
  • 举例：全称是 libnative-lib.so

编译可执行程序

1. 编写C语言

  • txt文件编写c代码，后转化成c文件

2. 编写android.mk代码

  • 和c文件在同一个目录下
  • android.mk文件：构建工具，用来告知ndk-build，最终被build System解析多次
  • 也就是说android.mk文件允许我们将我们的源文件打包成一个models，这个models可以是静态或者动态库，只有动态库才能被copy到我们的应用程序apk包里，静态库可以被链接到动态库里
  • 可以在android.mk里定义多个models，可以将同一份source加进多个models中

3. 编写Application.mk文件代码

  • 和c文件在同一个目录下

4. 编译.c文件

  • 需要使用ndk-build, 参考如何使用
