基本块概念

  • 基本块由一系列顺序执行的指令构成。每个基本块只有一个入口，就是开头第一句指令，也只有一个出口，就是结尾最后一句指令。执行时只能从其入口进入，从其出口离开。
  • 只要基本块中第一条指令被执行了，那么基本块内所有执行都会按照顺序仅执行一次。
  • 基本块在控制流混淆和反混淆方面有很重要的作用，它是控制流的基本概念和最小单元。基本块的定义本身就可以引申出一个基本的应用——如果我们只是想确认程序的执行流，比如程序在哪里死循环了，又或者去除虚假控制流，那么并不需要逐行汇编做 trace，只需要确认基本块的入口点被执行，就能确认整个基本块都会被执行，这种 trace 叫 block trace，相较于 code trace 开销要小很多，而且很多场景已经够用。

ARM32 APCS调用约定

  1. LR寄存器存放了程序的返回地址
  2. PC寄存器指向当前程序运行的位置。根据三级流水线，PC指向一条指令，其实是在执行此指令的上面第二个指令
  3. SP寄存器：余下的参数通过sp所指向的数据栈进行参数传递
  4. 当参数个数小于等于4个时，子程序间通过R0~R3来传递参数（即R0-R3代表参数1-参数4）
    5.如果参数个数大于4个，余下的参数通过sp所指向的数据栈进行参数传递。而函数的返回值总是通过R0传递回来。

ARM64 默认采用ATPCS调用约定

  1. ARM64 有34个寄存器，包括31个通用寄存器、SP、PC、CPSR
  2. 前八个参数通过 X0-X7 或 W0-W7 传递，多于8个的参数通过堆栈传递，因此sub_80D8的两个参数对应于X/W0和 X/W1
  3. x0-x7: 用于子程序调用时的参数传递和结果，X0还用于返回值传递。多余参数采用堆栈传递，64位返回结果采用X0表示，128位返回结果采用X1:X0表示
  4. x0-x30 是31个通用整形寄存器。每个寄存器可以存取一个64位大小的数。 当使用 x0 - x30 访问时，它就是一个64位的数。当使用 w0 - w30 访问时，访问的是这些寄存器的低32位
  5. X8：用于保存子程序返回地址， 尽量不要使用。
  6. X9-X15：临时寄存器，使用时不需要保存。
  7. X16 - X17：子程序内部调用寄存器，使用时不需要保存，尽量不要使用。
  8. X18：平台寄存器，它的使用与平台相关，尽量不要使用。
  9. X19 - X28：临时寄存器，使用时必须保存。
  10. X29：帧指针寄存器，用于连接栈帧，使用时需要保存。
  11. X30：链接寄存器 LR
  12. X31：堆栈指针寄存器 SP 或零寄存器 ZXR
