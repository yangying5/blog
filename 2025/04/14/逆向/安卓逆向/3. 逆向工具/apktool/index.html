ApkTool工具使用

下载

  • 下载地址：https://ibotpeaches.github.io/Apktool/
  • apktool版本汇总：https://bitbucket.org/iBotPeaches/apktool/downloads/
  • 注意：
    • 重命名下载的apktool_x.x.x.jar,改名为apktool.jar
    • apktool.jar 必须java环境才能运行
    • windows下设置环境变量可以使用

使用

  1. 解包apk获取资源文件：apktool d test.apk
  2. 重新打包apk：
    1. apktool b 解压后的项目目录
    2. apktool b 解压后的项目目录 -o 新apk的名字

使用经验记录

需要注意，如果回编译老是报错，且做了很多操作都没用，可能是某一个文件在第一次回编译错误后没有删除，即：删除这个文件C:\Users\Administrator\apktool\framework\1.apk

apktool反编译报错：Exception in thread “main” org.jf.dexlib2.dexbacked.DexBackedDexFile$NotADexFile: Not a valid dex magic value

  • 参考：https://blog.csdn.net/m0_37696990/article/details/103931261
  • 提示很明显，就是非法Dex，这是因为apk里有加密过后的dex文件，比如有些apk的assets目录下有加密后的Dex文件，添加–only-main-classes参数即可
  • apktool d ~/tmp/test.apk -o /tmp/test –only-main-classes

apktool.jar回编译报错：brut.androlib.AndrolibException: brut.common.BrutException: could not exec

  • 参考：https://blog.csdn.net/scimence/article/details/104998418

利用APKTOOL漏洞的apk会反编译失败，解决的话只能等apktool升级或找app的老版本

  • 使用新版本的时候，下载最新版本的apktool.jar包；
  • 然后使用命令：java -jar .\apktool_2.4.1.jar d -f “***.apk”

回编译报错，可添加命令–use-aapt2

  • 如：.\apktool.bat d –use-aapt2 .\du.apk

Exception in thread “main” brut.android.AndrolibException: Multiple res specs: attr/name

  • 主要是因为利用了apktool的漏洞做了属性id的混淆，导致在反编译过程中存入了重复的id值
  • 错误代码：RestypeSpec.java的addResSpec方法78行
  • 修复： 在这个方法存入map数据之前做一个判断操作即可，判断属性id是否存在这个map，存在则直接访问

Exception in thread “main” brut.android.AndrolibException: Could not decode arsc file

  • 主要是利用apktool的漏洞，修改了resource.arsc的头部信息，导致在分析头部数据结构时出错
  • 错误代码：ExtDataInput.java的skipCheckChunkTypeInt方法第73行
  • 修复：修复resource.arsc头部数据，修改skipCheckChunkTypeInt检测方法逻辑

brut.androlib.AndrolibException: brut.common.BrutException: could not exec (exit code = 1)

  • 参考：https://www.jianshu.com/p/5d213fc2ed73
  • java -jar apktool.jar d test1.apk -o test1
  • java -jar apktool.jar b test1
    改为
  • java -jar apktool.jar -r d test1.apk -o test1
  • java -jar apktool.jar b test1
