方式1：单个处理[非批量]

参考

  • https://bbs.pediy.com/thread-259062.htm
  • http://www.yxfzedu.com/rs_show/150

技术要点

  • 遇见JUMPOUT就patch
  • ida会帮我们解析出一些跳转的偏移，它解析不了的，我们需要帮助它
  • 记得删除函数块和新建函数块，同时有些时候你需要帮助ida确定函数尾
  • 我的代码不能patch所有地方，每遇到一个JUMPOUT，你需要修改偏移，然后手动patch（因为它们的JUMPOUT特征不固定，想完美patch，需要完美的patch代码，哈哈，不过它一个函数JUMPOUT的地方还不是很多，你按我的方法做也不累）
  • 至于fla嘛，你随意吧，网络上对抗fla的帖子还是挺多的，另外想动态调试可以忽略它

patch代码

  1. 修改patch代码中的地址后让ida运行此python脚本
  2. 给鼠标放在代码块上，点ida的edit-functions-delete function；
  3. 在代码块起始位置按p，之后按f5
    注意： 如果edit-functions后面的按钮按不动，可以给代码从c语言切换到汇编代码

方式2：idapython & keypatch[批量]

keypatch使用

  • Keypatch
  • 大部分绿色版 IDA 都自带 Keypatch
  • Patcher 是修改某一行汇编为指定汇编；
  • Fill Range 是先选定一片区域，然后将这片区域修改为指定汇编代码，它的使用场景之一是批量 nop ；
  • Undo last patching 是撤销前一次做所的 patch 操作

Patch 后的重新分析

并不会立刻发生什么改变，比如对 JNI_OnLoad 重新 F5，依然是 jumpout。因为我们还没有让 IDA 重新分析——根据我们 patch 后的新现状做重新分析<可以选择 options - general 子菜单，切换到 Analysis 窗口，点击重新分析程序按钮，并确认 OK，即可实现程序的重新分析; 快捷键右键左下角；实际使用中你会发现它并不是真正、全盘、彻底的重新分析>

Patch 后的持久化

首先让 Patch 持久化：Edit--Patch program--apply patches to input_file
将我们所作的 patch 保存回原文件。关闭 IDA（不保存任何 IDB 文件），重新打开 patch 后的 SO

搜索与匹配

  • 思路总结：搜索匹配可以根据助记符的顺序 | 固定二进制 | 最好反汇编text段，遍历地址，判断是否调用目标方法或者方法集 | 交叉引用<少用，因为花指令存在，因为它自身不一定被顺利识别为函数，调用它的地方不一定被正确识别为代码，也就不存在对应的引用信息> | junkCode的内容<然后遍历text段，每四个字节判断是否是B跳转指令且跳转地址在junkCode的match列表>
  • 因为花指令的存在，最好使用纯粹的反汇编<如idapython | capstore>对样本text段进行反汇编

搜索：适用于特征明显，可用二进制直接匹配类型花指令

  • 二进制搜索与汇编搜索<在 IDA 中十分值得一提的还有 AST Search>

    搜索汇编: 菜单栏 Search - Text 可以进行汇编搜索，记得勾选 Find all occurences，寻找所有出现地方

二进制搜索: 首先打开机器码展示，设置为 4 字节, 菜单栏Search - Sequence of bytes 做二进制搜索

我们在处理花指令时，绝不会使用汇编匹配。因为花指令本就用于混淆和干扰反汇编过程，它很可能会让 IDA 将某些代码误识别为数据，以及某些数据误识别为代码

二进制搜索，它的好处就是不存在漏判的问题，但容易误判，即某些数据可能恰好和机器码的值相同。因此，在二进制搜索时需要更严格的匹配规则，比如从搜索四个字节调整为搜索八个字节

总结：样本中常见的会是两种方式的结合使用

计算实际跳转地址

方式1： 根据具体逻辑，使用idaapi获取操作数或者capstone反汇编获取操作数

IDA 的idc.get_operand_value(ea, n)API 可以很方便的获取ea地址处汇编代码的第n个操作数的值（n 从 0 开始计数）

因为花指令等存在，代码未被ida识别，以数据块形式展示，因此无法解析藏在数据块里的花指令，如何做？
  1. 一个比较流行的方案是通过 IDAPython 获取 junk code 对应的 bytes，然后放到 Capstone 这样的第三方反汇编引擎中解析为汇编代码，而不依赖 IDA 自己的反汇编模块
  2. 将匹配到的位置通过 API 识别为代码

方式2：flare-emu / Unicorn

可以让 Flare-emu 从首地址开始执行，一直运行到 目标寄存器 生成的地方，并获取 目标寄存器 的值。
junk code 往往是一串不复杂的汇编代码，不复杂指其中基本不包括外部函数调用或系统调用，所以使用模拟器很方便

Patch

  1. 自写这条 B 指令所对应的汇编逻辑，生成B xxx所对应的机器码
  2. 使用成熟的汇编引擎，比如 Keystone，上面使用的 Keypatch 其底层就是 Keystone
  • 确保 keystone 成功安装后调用 API 运行脚本后，将 patch 持久化 保存回 SO 重新打开
